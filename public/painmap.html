CORE STRATEGY

고객을 어떻게 알아보고, 안 귀찮게 데이터를 모을까

고객은 회원가입을 싫어한다. 마찰을 0에 가깝게 — 전화번호 하나가 만능 키다. 도수 이력은 전화번호+이름으로 조회되는 게 업계 표준. 여기서 출발한다.

① 전화번호 = 식별 키방문/예약 시 전화번호만. 기존 고객이면 도수·방문횟수·선호 즉시 팝업. 신규는 번호 1개로 시작.
② 카카오싱크 1탭회원가입 폼 대신 QR → 채널 친구추가 1탭. 가입 마찰 제거 + 식별 + 마케팅 동의 동시.
③ 알림톡 (정보성)전화번호+비즈채널이면 발송. 예약확인·준비완료·1년 재검 리마인드. 마케팅 동의 없이도 리텐션.
④ 친구톡 (광고성)채널 친구 + 수신동의 후. 신상·프로모션·세그먼트 타겟. 카톡이 마케팅 본진.

⚠️ 경쟁 참고 — 레티나(안경업계 점유율 50%)가 이미 고객관리·검안이력·고객모니터·본사 대시보드를 함. BETTERVISION은 CRM을 새로 발명하는 게 아니라 본부 데이터 귀속 + 9사이즈 측정표준 + 가상피팅 + 카톡 마케팅으로 차별화·통합해야.

사이즈 맞는지 일일이 써봐야 함
해법: 카메라로 사이즈 측정. ✅ measure.html 안면측정(카메라)·9사이즈 환산 있음.

2
조용한 매장 뻘쭘 · 접객 부담 · 몇 개 못 써보고 삼
해법: 가상 추천·피팅으로 부담 제거. ✅ recommend 가상피팅 + ✅ manual 접객강도 가이드(무관심도 과잉도 아니게).
접객 강도 가이드(말 걸 타이밍·거리감) 매뉴얼 — 무관심도 과잉도 아니게
고객이 폰으로 미리 골라오면 접객은 확인·업셀만
3
어떤 모델 있는지 모름 · 가서 봐야
해법: 전 모델 온라인 게시 + 사이즈 매치·벌림각 커버 판정. ✅ catalog (PB 9사이즈 제공 + 메이커 매치%·벌림각 판정).
모델별 내 사이즈 매치% + "벌림각 조정으로 커버 가능/불가" 판정 표시
4
렌즈 가격·정보 안 나옴 · 즉흥 결정
해법: 렌즈 가격·특성 기재. ✅ pricing 렌즈 굴절률×기능 가격표.
렌즈 카탈로그 — 등급(굴절률·코팅·기능)별 가격·특성 투명 공개
5
눈에 잘 맞는지 확신 안 섬 · 안경사 신뢰
해법: 검안 시스템·안경사 신뢰도로 확신. ✅ exam + ✅ trust(면허·장비·프로세스 투명).
검안 과정·정밀도 고객 설명(시스템 투명화) + 안경사 면허·경력 프로필
6
콘택트 가격·색·특징 안 나옴
해법: 콘택트 가격·특징 모델별 기재. ✅ pricing 콘택트 카탈로그 + ✅ shop 구매(🔓 샌드박스 가정).
콘택트 카탈로그 — 모델·색·도수범위·가격·특성. 온라인 판매는 샌드박스 통과 후
7
눈탱이(바가지) 걱정
해법: 눈탱이 안 치게 강조. ✅ pricing 표준가 보증 + ✅ quote 견적서 자동 발급.
본부 표준가 고정·노출 — 가맹점 임의 인상 불가(서버 강제), 정가 공개
견적서 자동 발급 — 프레임+렌즈+옵션 명세, 결제 전 투명
렌즈 등급별 가격표 공개 · 후기/평점 · "표준가 보증" 배지
8
메이커 안경 어디 있는지 모름
해법: 제휴 메이커 가격·사이즈 + 내 사이즈 매치. ✅ catalog 메이커 매치%·벌림각 판정(실데이터 입력 필요).
취급 메이커 온라인 게시(가격·사이즈) + 모델별 내 사이즈 매치 판정
9
재구매 매번 방문·검안 번거로움
해법: 고객 데이터 저장 + 검안 트래킹. ✅ profile 통합(도수 이력·시력추세·재구매).
통합 프로필에 도수·사이즈 저장 → 재구매 원클릭(검안 생략 가능 여부 안내)
검안 트래킹(시력 변화) → 재검 필요 시만 안내
10
선글라스는 온라인으로 사고 싶음
해법: 선글라스 온라인 + 도수 재구매·콘택트(샌드박스). ✅ shop(선글라스=합법 / 도수재구매·콘택트=🔓 샌드박스 가정·표시).
선글라스 온라인 스토어(비도수=합법) — 치수·추천·결제·배송
도수 안경 재구매는 규제 샌드박스 신청 → 픽업/온라인 배송 트라이 (콘택트도 샌드박스 후)
가맹점 9 페인포인트 × OS 매핑
1
접객 시스템 — 누가 왔는지 자동으로

네 해법

기존 OS

△ flow.html 접객 시퀀스(허브)는 있으나 예약 연동·재방문 자동 인식은 없음.

새로 만들 것

예약→방문 시 고객카드 자동 팝업 (전화번호 인식 → 방문횟수·지난 도수·선호 프레임)
신규/단골/VIP 자동 배지 + 접객 가이드 분기
예약 캘린더(알림톡 예약확인 연동)
2
손님 데이터 — 어떻게 쌓을지

네 해법

CRM으로 잘 쌓기. 고객 최대한 안 귀찮게.

기존 OS

△ customer.html 멤버십·포인트 + bv-record 측정레코드는 있으나 통합 고객 프로필/타임라인은 없음.

새로 만들 것

저마찰 수집(카카오싱크·전화번호) — 위 CORE 전략

기존 OS

네 해법

새로 만들 것
