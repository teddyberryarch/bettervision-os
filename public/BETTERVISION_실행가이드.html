순서대로 실현한다

단일 진실 공급원(Single Source of Truth) ｜ 양지근 ｜ 2026.06
문제는 "측정 시스템 만들기"가 아니라 흩어진 아이디어를 한 곳에 모아 실제로 실현하는 것이다. 메모리는 꽉 찼고, 아이디어는 여러 채팅에 흩어졌다. 해법은 하나 — 이 문서를 단일 진실 공급원으로 삼는다. 메모리·채팅은 보조, 진짜는 이 문서. 새 채팅 켤 때마다 이 파일만 던지면 맥락이 복원된다.
▶ 새 채팅으로 갈아탈 때 (이대로만 하면 됨)
  • 1. 새 채팅을 연다.
  • 2. 이 파일(실행 가이드) + 마스터 설계도 v0.2 docx를 같이 올린다.
  • 3. "BETTERVISION 실행 가이드의 [N단계] 이어서 하자"라고 말한다.
  • 4. 끝. 그 채팅에서 진행한 결과를 이 문서에 누적 업데이트한다.

이렇게 하면 메모리가 꽉 차도, 채팅이 바뀌어도 안 흩어진다. 문서가 본체다.

A실현하는 법 (원리) B타임라인 — 0~4단계 실행 순서 C전체 아이디어 인덱스 (뭐가 있나) D파일 지도 — 마스터 인덱스

A. 실현하는 법 — 원리

핵심 진단: 지금 필요한 건 아이디어를 더 모으는 것도, 또 만드는 것도 아니다. 흩어진 걸 한 곳에 모아 실행 가능하게 고정하는 것.

병목은 "만들기"가 아니라 "출시·과금"

AI 시대엔 만드는 비용이 폭락했다. 진짜 병목은 만든 걸 사람 앞에 내놓고 돈 받느냐. 그래서 실행 기준선을 '만들기'가 아니라 '첫 데모 → 첫 펀딩 → 첫 매장'으로 잡는다.

3가지 저장소의 역할 분담

저장소역할
이 문서 (정본)단일 진실 공급원. 무엇을·어떤 순서로·어디까지 했나. 여기가 본체.
마스터 설계도 v0.2'무엇을 만드나'의 상세 설계(①~⑤·피팅·검안). 이 문서와 짝.
메모리 / 채팅보조. 압축 인덱스·작업 로그. 꽉 차도 문서가 있으니 괜찮음.

B. 타임라인 — 0~4단계 실행 순서

측정 엔진 하나로 데모 → 펀딩 → 매장 검증. 앞 단계가 뒤를 쉽게 만든다.

0단계 · 지금 바로

측정 데모 앱 진행 예정

기간 1~2주 · 가장 레버리지 큼
  • 아이패드 1대로 측정 5값(PD·얼굴폭·코높이·귀위치·안면각) → 9사이즈 코드 환산 → 가상피팅 → DB 저장.
  • 페이스룰러 API 붙여 "측정→매칭→피팅→DB"가 한 흐름으로 돌아가는 걸 보이는 게 설득의 90%.
  • 이게 펀딩·가맹·팀 설득의 핵심 무기.
1단계 · 병행

데이터 표준 확정 설계 완료, 구현 대기

기간 2~3주
  • raw 3층 구조 + 9사이즈 환산 규칙 1차 버전(양지근 면허 기준).
  • 경계 mm값 임시 확정 → 나중에 실데이터로 자기교정.
  • 동의·보안(생체정보) 필드 처음부터 박기.
2단계

② 타사 안경 DB 시드 + 큐레이팅 대기

기간 1개월
  • 진열할 타사 안경 각인(52□18-145) 긁어 DB화. 소재→휨한계 테이블 같이.
  • 적합도 % 매칭 데모("이 젠몬 78% vs PB 96%").
  • 진열=체험=데이터 시드 동시. 타사도 본사 소싱 락업(법무 검토).
3단계

A4 피팅시트 + ③ 검안 표준화 대기 (검안변수=다음 핵심)

기간 1개월
  • 피팅시트 실사용 버전 완성(이미 데모 있음).
  • ③ 검안 변수 체계 잡기 — 안경광학 교과(굴절검사·양안시·교정·조제가공·재료·기하광학)를 표준 목차로, 단계별 AI 기준값·정상범위·이상치 정의. 양지근 임상지식 필요.
  • 화면 촬영+AI 판독으로 기존 검사장비 값 받기 시작.
4단계

펀딩 · 안테나샵 대기

기간 3~6개월
  • 데모 패키지로 초기창업패키지(2027 Q2 최종창)·투자 어필.
  • 성수 안테나샵 자력 오픈 → 6개월 실데이터 → OS 검증 → 밸류 점프 후 본격 투자.
  • 가맹 1+1 요건(실제 소매 운영 1년)은 아버지 매장 전환으로 충족 검토.
한 줄 순서: 측정엔진 데모(0) → 데이터표준(1) → 타사DB·큐레이팅(2) → 피팅·검안(3) → 펀딩·매장(4)

D. 파일 지도 — 마스터 인덱스

파일은 두 뭉치다. 헷갈리면 여기만 본다. 정본(★)이 어느 쪽인지 표시.

왜 두 뭉치인가

① OS 앱 = 실제 돌아가는 데모 프로그램(public/ 폴더, Railway 배포). ② 문서 = 폰서 읽는 설계·전략 정본. 같은 내용이 양쪽에 있으면 앱판이 정본, 문서판은 스냅샷.

① OS 앱 — bettervision-os/public/

상단 네비 8개 = 홈·사업개요·고객앱·가맹점·가맹본부·접객·검안·피팅. 접객(flow.html)이 전체 허브 — 맨 아래 시스템 맵에서 측정·추천·엔진 등 도구 전부 링크.
